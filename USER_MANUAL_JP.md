# 虎威連携用 GAS セットアップマニュアル

このマニュアルでは、GitHub Release から `code.js` を取得し、自分の Google スプレッドシートに貼り付けて、虎威と連携するまでの手順を説明します。

## 事前に用意するもの

- Google アカウント
- 虎威で使用する Google スプレッドシート
- GitHub Release ページから取得する `code.js`
- 虎威のログインアカウント

## 1. Google スプレッドシートを開く

1. 虎威と連携したい Google スプレッドシートを開きます。
2. 投稿管理に使用するシートを用意します。
   - `Posts`
   - `Posted`
   - `Errors`

「Google スプレッドシートを開いた画面をキャプチャ」

## 2. Apps Script エディタを開く

1. スプレッドシート上部メニューの `拡張機能` をクリックします。
2. `Apps Script` をクリックします。
3. 新しいタブで Apps Script エディタが開きます。

「拡張機能 > Apps Script を開く画面をキャプチャ」

## 3. GitHub Release から code.js を取得する

1. X_Autopost の GitHub Release ページを開きます。
2. 最新の Release を開きます。
3. Assets から `code.js` を開く、またはダウンロードします。
4. `code.js` の内容をすべてコピーします。

「GitHub Release の Assets に code.js が表示されている画面をキャプチャ」

## 4. Apps Script に code.js を貼り付ける

1. Apps Script エディタで `コード.gs` または既存のコードファイルを開きます。
2. 既存のコードをすべて削除します。
3. GitHub Release からコピーした `code.js` の内容を貼り付けます。
4. 画面上部の保存ボタンをクリックします。

保存ボタンを押さないと、貼り付けたコードが反映されません。必ず保存してください。

「Apps Script エディタに code.js を貼り付けた画面をキャプチャ」

「Apps Script の保存ボタンをキャプチャ」

## 5. appsscript.json を確認する

1. Apps Script エディタ左側の `プロジェクトの設定` を開きます。
2. `appsscript.json マニフェスト ファイルをエディタで表示する` を有効にします。
3. 左側のファイル一覧から `appsscript.json` を開きます。
4. Release に `appsscript.json` が同梱されている場合は、その内容を貼り付けて保存します。

Drive のメディア投稿を使用する場合、Drive API など必要な権限が manifest に含まれている必要があります。

「プロジェクト設定で appsscript.json 表示を有効にする画面をキャプチャ」

「appsscript.json を開いた画面をキャプチャ」

## 6. Web アプリとしてデプロイする

1. Apps Script エディタ右上の `デプロイ` をクリックします。
2. `新しいデプロイ` をクリックします。
3. 種類の選択で `ウェブアプリ` を選びます。
4. 以下のように設定します。
   - 説明: 任意の説明を入力
   - 次のユーザーとして実行: `自分`
   - アクセスできるユーザー: `全員`
5. `デプロイ` をクリックします。
6. Google から権限確認が表示された場合は、内容を確認して許可します。

「デプロイ > 新しいデプロイを開いた画面をキャプチャ」

「種類の選択でウェブアプリを選ぶ画面をキャプチャ」

「ウェブアプリの設定画面をキャプチャ」

「Google の権限確認画面をキャプチャ」

## 7. GAS Web アプリ URL を取得する

1. デプロイが完了すると、`ウェブアプリ URL` が表示されます。
2. 表示された URL をコピーします。
3. URL は通常 `/exec` で終わります。

この URL は虎威のプロフィール/API設定画面で使用します。

「デプロイ完了後にウェブアプリ URL が表示されている画面をキャプチャ」

## 8. スプレッドシートを再読み込みする

1. 元の Google スプレッドシートのタブに戻ります。
2. ブラウザでスプレッドシートを再読み込みします。
3. 上部メニューに `虎威連携` が表示されることを確認します。

`虎威連携` メニューが表示されない場合は、Apps Script のコード保存やデプロイが完了しているか確認してください。

「スプレッドシート上部に虎威連携メニューが表示された画面をキャプチャ」

## 9. 本人確認コードを取得する

1. スプレッドシート上部メニューの `虎威連携` をクリックします。
2. `本人確認コードを生成` をクリックします。
3. 表示された本人確認コードをコピーします。

本人確認コードの有効期限は 10 分です。期限が切れた場合は、もう一度 `本人確認コードを生成` を実行してください。

「虎威連携 > 本人確認コードを生成をクリックする画面をキャプチャ」

「本人確認コードが表示されたダイアログ画面をキャプチャ」

## 10. 虎威に GAS URL と本人確認コードを保存する

1. 虎威にログインします。
2. プロフィール/API設定画面を開きます。
3. `Google Sheets URL` に、手順 7 で取得した Web アプリ URL を入力します。
4. `GAS本人確認コード` に、手順 9 で取得した本人確認コードを入力します。
5. `保存` をクリックします。

保存に成功すると、虎威とスプレッドシートの本人確認が完了します。

「虎威のプロフィール/API設定画面をキャプチャ」

「Google Sheets URL と GAS本人確認コードを入力した画面をキャプチャ」

## 11. 本人確認完了後の表示

本人確認が完了すると、虎威側では GAS URL が表示専用になります。GAS URL を変更する場合は、`変更` ボタンを押してから、新しい Web アプリ URL と新しい本人確認コードを入力してください。

「本人確認完了後、GAS URL が入力不可になっている画面をキャプチャ」

## 12. よくある確認ポイント

### 虎威連携メニューが表示されない

- Apps Script に `code.js` を貼り付けたあと、保存ボタンを押したか確認してください。
- スプレッドシートを再読み込みしてください。
- 正しいスプレッドシートに Apps Script を設定しているか確認してください。

### 本人確認コードを入力しても失敗する

- 本人確認コードの有効期限 10 分を過ぎていないか確認してください。
- コードをコピーするときに空白が入っていないか確認してください。
- Apps Script を新しいバージョンとしてデプロイしているか確認してください。
- 虎威に入力した URL が `/exec` で終わる Web アプリ URL か確認してください。

### 古いコードが動いているように見える

- Apps Script に貼り付けたあと、保存ボタンを押したか確認してください。
- `デプロイを管理` から、Web アプリが最新バージョンになっているか確認してください。
- 必要に応じて新しいバージョンとして再デプロイしてください。

## 13. 更新時の手順

GitHub Release で新しい `code.js` が公開された場合は、以下の手順で更新します。

1. 最新 Release から `code.js` を取得します。
2. Apps Script エディタの既存コードをすべて置き換えます。
3. 保存ボタンをクリックします。
4. `デプロイを管理` から Web アプリを編集します。
5. バージョンで `新バージョン` を選択します。
6. デプロイします。
7. 必要に応じてスプレッドシートを再読み込みします。

「デプロイを管理で新バージョンを選択する画面をキャプチャ」
